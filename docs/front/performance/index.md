## 方向

性能优化是一个纬度比较大的方向

可以从http、项目本身以及加钱解法（增加服务起配置）几个方面


### http

- dns预解析
- http2.0
- http缓存



### 项目

- 降低http请求数

浏览器允许同时并发请求数目是有限制的（一般4-7个？），因此，后边的资源需要等待前面的资源加载后才能开始加载（阻塞）。


- 精灵图 
- webpack、vite等打包工具合并资源（小图base64）
- 路由懒加载
- 组件懒加载
- 图片懒加载
- cdn分发
- 开启gzip
- 服务端渲染
